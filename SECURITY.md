# Security Policy

## Supported Scope

Security-sensitive areas include:

- `services/api/`
- `services/collab-relay/`
- `apps/web/public/assets/appcheck.js`
- `apps/web/public/assets/modules/` (modular editor)
- Firebase rules and hosting configuration

## Reporting a Vulnerability

Please do not open public issues for vulnerabilities that could expose users, spaces, or infrastructure.

Report privately to:

- `info@malzi.space`

Include when possible:

- affected area or file
- reproduction steps
- impact assessment
- logs or screenshots if safe to share
- suggested fix if available

## Response Goals

Target process:

1. acknowledge receipt
2. validate impact
3. mitigate or patch
4. deploy and verify
5. document follow-up if needed

Operational details for maintainers live in `docs/security/SECURITY_RUNBOOK.md`.
